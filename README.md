# 🀄 日麻三麻進階訓練場 (Sanma Advanced Trainer)

一個專為日本麻將（三人麻將）愛好者打造的進階 Web 訓練與競技平台。
整合了「防守權重分析」、「AI 模擬對局」與「即時多人連線錦標賽 (Duplicate PK)」，旨在幫助玩家透過科學化的數據與實戰演練，提升攻防判斷能力。

## ✨ 核心特色 (Key Features)

### 🎮 全新沉浸式遊戲大廳 (Immersive Game Lobby)
* **卡片式大廳介面**：移除傳統分頁導覽，新增大型卡片式的「遊戲大廳」模式選擇介面，並配備動態背景與懸停特效，大幅提升整體應用程式的遊戲客戶端沉浸感。
* **防呆退出機制**：新增導覽列「返回大廳」按鈕，並完美整合錦標賽中途退出的防呆棄權警告機制，避免玩家誤觸退出比賽。

### 🏆 複式競技賽 (Duplicate PK)
採用「複式麻將 (Duplicate Mahjong)」概念，考驗玩家在相同進張下的攻防選擇。
* **公平競技**：所有參賽者在相同的牌山種子 (Seed) 下分別與 AI 對戰，解決麻將運氣成分問題。
* **全螢幕觀戰模式 (Spectator Mode)**：錦標賽對局提早結束後，玩家可點擊雷達面板上的觀戰按鈕，以「上帝視角」即時觀看其他玩家的牌桌狀態與捨牌池。
* **即時對戰雷達 (PK Radar)**：透過 Firebase 即時同步，隨時掌握對手的巡目、立直狀態與思考時間。
* **高可靠性與便利性**：等候室支援一鍵複製房間代碼；連線中支援斷線自動棄權、房主權限自動繼承 (Host Migration) 以及回合結束自動倒數推進。
* **新手輔助支援**：多人模式亦解鎖完整的「聽牌預測看板」與手牌上方的「聽牌/破聽」狀態標籤。

### 🤖 深度實戰模擬引擎 (Mahjong Engine)
核心邏輯位於 `src/engine/MahjongEngine.js`，提供精確的判斷支援：
* **防守權重引擎**：根據敵方立直與鳴牌狀態，動態計算手牌的「危險度分數」與進張效率。
* **戰術立場建議**：結合《數據制勝》理論，提供即時攻防建議。AI 防守邏輯動態調整，當面對玩家立直且自身已聽牌時將強制對攻，增加對戰樂趣。
* **精準算番系統**：支援 DFS 遞迴算番、一砲雙響、流局罰符，以及三麻特有的「拔北」寶牌與裏寶牌機制。

### 🎵 沉浸式音訊與 QoL (Immersive Audio & Quality of Life)
* **生熟張高亮輔助**：在單機練習與觀戰模式中，滑鼠懸停於手牌時，三家河牌池與副露區的同款牌會瞬間亮起黃色光圈，大幅降低尋牌負擔。
* **直覺化結算畫面**：寶牌與裏寶牌指示由系統代碼全面轉換為直覺的完整中文牌名 (如：赤5筒)。
* **動態音訊系統**：實裝摸切、鳴牌、立直閃電及和牌爆發音效；BGM 會根據大廳、一般對局與「立直激戰」情境無縫切換。
* **自訂設定**：提供獨立音訊設定面板，支援 BGM 與 SFX 獨立音量調整並自動記憶偏好。

## 🏗️ 專案架構與檔案分析 (Architecture Analysis)
本專案採用 React 19 + Vite 6 構建，狀態管理採用純 React Hooks，並遵循「邏輯與表現分離」原則。

📂 目錄職責詳解：
* `src/assets/`: 靜態資源、圖示與遊戲音效檔 (.mp3)。
* `src/components/`: 🎮 共用 UI 元件 (如 `Tile.jsx` 支援危險度標記，`PKRadar.jsx` 支援即時對戰雷達)。
* `src/constants/`: ⚙️ 靜態常數 (如：麻將牌組定義、預設值)。
* `src/data/`: 📖 靜態資料庫 (如 `changelogData.js` 版本更新與 `tacticsData.js` 局收支題庫)。
* `src/engine/`: 🧠 核心邏輯引擎 (純 JS 邏輯，掌管向聽數、理牌、算番、危險度評估)。
* `src/hooks/`: 🎣 自定義 Hooks (狀態管理核心，包含單局模擬 `useSimulation` 與 Firebase 即時監聽 `useTournamentSync`)。
* `src/services/`: 🌐 外部服務 API (封裝 Firebase 建立房間、分數上傳等操作)。
* `src/utils/`: 🛠️ 工具函式 (Firebase 初始化設定與 RNG 亂數生成器)。
* `src/views/`: 🖥️ 主要頁面視圖 (包含大廳路由、實戰模擬沙盒、錦標賽大廳、戰術學院與術語百科)。

## 🔄 資料同步設計 (Data Flow)
* **單機沙盒**：完全依賴 `useSimulation` 在 Client-side 運作，結合 `MahjongEngine` 實現零延遲對弈。
* **錦標賽連線**：
    * **Seed 共享**：透過共用 RNG Seed，確保在無 Server 端發牌情況下，各 Client 生成一致的牌山。
    * **即時推播**：摸切動作觸發內部結算後，透過 `tournamentService` 將分數或 Live 動作同步至 Firebase。
    * **即時監聽**：`useTournamentSync` 負責監聽 Firebase 更新，渲染 PK 雷達、排行榜，並支援實時全螢幕觀戰系統。

## 🛠️ 維護與除錯指引 (Debugging Guide)
* **邏輯除錯**：若發現和牌判定、算番或牌效有誤，應優先檢查 `src/engine/MahjongEngine.js`，此檔案不依賴 React 狀態。
* **同步異常**：若錦標賽中對手數據未更新或觀戰畫面異常，請檢查 `useTournamentSync.js` 的監聽邏輯與 Firebase 資料庫權限。
* **狀態流轉**：遊戲操作卡死（如無法摸牌）通常與 `useSimulation.js` 中的 `gameState` 鎖定有關。
* **RNG 一致性**：若發生玩家牌山不一致，請檢查 `src/utils/rng.js` 的 `seedString` 是否正確傳遞至各個客戶端。
